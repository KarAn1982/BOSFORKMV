# БОСФОР — исполняемый прототип API заявок

Дата: 19 июня 2026 года

## Назначение

`prototype_server.js` позволяет проверить полный путь формы до серверного ответа
до выбора production-хостинга и CRM.

Запуск:

`node prototype_server.js`

Адрес по умолчанию: `http://127.0.0.1:4173`.

При таком запуске `runtime-config.js` автоматически включает `/api/leads`.
При обычном открытии статических HTML endpoint отсутствует и интерфейс остаётся
в безопасном демонстрационном режиме.

## Реализовано

- `multipart/form-data`;
- обязательные `form_id`, `audience`, consent, page URL и телефон/email;
- проверка Origin и page URL;
- honeypot;
- минимальное время заполнения;
- rate limit по хешу IP и контакта;
- обязательный `Idempotency-Key`;
- ограничения количества и размера файлов;
- whitelist расширений;
- сигнатуры PDF, ZIP/DOCX/XLSX, DOC/XLS, DWG и DXF;
- тестовая блокировка EICAR-строки;
- in-memory retry queue без контактных данных;
- события `lead_submit_success` и `lead_submit_error`;
- пользовательские success/error состояния;
- 404 и health endpoint.

## Что намеренно не считается production-готовым

- тест EICAR не заменяет ClamAV или облачный антивирус;
- ZIP не распаковывается и не проверяется на archive bomb;
- очередь хранится только в памяти и содержит только безопасные метаданные;
- нет шифрованного объектного хранилища;
- нет реальной CRM и повторной доставки;
- rate limit должен быть перенесён в Redis/edge storage;
- секреты, аудит и alerting настраиваются в production-инфраструктуре.

## Проверка

`test_prototype_api.js` тестирует:

- health;
- слишком быструю отправку;
- валидный PDF и постановку в очередь;
- idempotency;
- запрещённое расширение;
- поддельную сигнатуру PDF;
- honeypot и чужой Origin;
- тестовую вредоносную сигнатуру;
- лимиты по контакту и IP;
- browser success/error и аналитические события.
