# БОСФОР — рекомендуемая техническая архитектура

Дата решения: 19 июня 2026 года

## Решение

Рекомендуемый стек первого production-релиза:

- Next.js с серверным и статическим рендерингом;
- Payload CMS;
- PostgreSQL;
- S3-совместимое хранилище изображений и документов;
- CDN перед приложением и медиаконтентом;
- интеграция форм через серверный endpoint с CRM;
- Яндекс Метрика и нейтральный `dataLayer`;
- deployment через staging и production environments.

Версии фреймворков фиксируются перед началом разработки после проверки
поддерживаемых стабильных релизов.

## Почему этот вариант

- уникальный дизайн без зависимости от page builder;
- единый TypeScript-код для сайта и CMS;
- структурированные модели услуг, проектов, статей и экспертов;
- серверный HTML для SEO и AI-search;
- автоматическая генерация canonical, Open Graph, JSON-LD и sitemap;
- контроль индексируемых URL;
- возможность поэтапно развивать личные кабинеты, калькуляторы и интеграции.

## Альтернатива

WordPress с собственным block theme и ACF допустим, если приоритетом станет
минимальная стоимость поддержки. Не использовать Elementor, WPBakery и
одновременное сочетание нескольких page builder: они ухудшат управляемость,
скорость и консистентность structured data.

### Шаблоны страниц

- главная;
- аудитория;

- статья;

## Рендеринг

- главная, услуги, проекты и статьи — статический HTML с ревалидацией;
- формы — серверные действия или API endpoint;
- фильтры проектов — клиентское состояние без создания индексируемых
  параметрических дублей;
- preview CMS — только в закрытой staging-среде;
- поиск — отдельный endpoint, закрытый от индексации.

## Формы

Поток данных:

`браузер → server validation → anti-spam → CRM → уведомление → аналитика`

Обязательные меры:

- серверная валидация;
- rate limit;
- honeypot;
- журналирование статуса передачи;
- повторная доставка при временной ошибке CRM;
- запрет загрузки исполняемых файлов;
- проверка размера и MIME-типа;
- отдельный consent timestamp.

## SEO-автоматизация

CMS-шаблон формирует:

- title и description;
- canonical;
- Open Graph и social image;
- breadcrumbs;
- `Organization`, `Service`, `Project`, `Article`, `FAQPage`;
- XML sitemap и image sitemap;
- `noindex` для preview, поиска, технических страниц и параметрических дублей.

## Среды

### Staging

- закрыта basic authentication;
- `noindex, nofollow`;
- отдельные тестовые CRM и аналитика;
- preview контента;
- визуальная регрессия перед релизом.

### Production

- автоматический backup базы;
- versioned media;
- мониторинг 404/500 и формы;
- CDN и кеширование;
- журнал деплоев и возможность отката.
