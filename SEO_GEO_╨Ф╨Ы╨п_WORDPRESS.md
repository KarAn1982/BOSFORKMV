# БОСФОР — SEO/GEO при переносе в WordPress

SEO/GEO уже заложен в прототипы. Задача при переносе — **не потерять его**, а не строить заново. Ниже что сохранить и как настроить в WP.

## 1. Мета по страницам — взять точь-в-точь
Файл `SEO_META_ПО_СТРАНИЦАМ.csv` — для всех 45 страниц: title, meta description, canonical, типы schema. Для каждой WP-страницы вставь в SEO-плагин (Rank Math или Yoast) тот же title и description из таблицы. Ничего не переписывай «на глаз» — это уже оптимизировано.

## 2. URL (слаги) — ОЧЕНЬ ВАЖНО
Имена файлов прототипов английские (`private-clients.html`), но **боевые URL должны быть человекочитаемыми ЧПУ**. Ставь слаг WP-страницы по этому списку (из архитектуры):

## 3. Schema (JSON-LD) — сохранить разметку
На контентных страницах в прототипах есть JSON-LD (типы Service, Organization, WebPage, Article и т.д. — см. колонку в CSV). Два пути:
- проще: включить генерацию schema в Rank Math (Organization/LocalBusiness глобально + тип на страницу);
- точнее: скопировать готовый JSON-LD из прототипа на соответствующую страницу (блок «Произвольный HTML»).
После настройки проверить: search.google.com/test/rich-results.

## 4. Корневые файлы (папка `seo-geo-файлы/`) — положить в корень сайта
- **`robots.txt`** — уже с разрешением ИИ-краулеров (`OAI-SearchBot`, `GPTBot`) — это и есть GEO. В WP заменить виртуальный robots на этот (или плагином). Проверить, что в нём указаны обе карты сайта.
- **`llms.txt`** — загрузить в корень как есть (`https://bosforkmv.ru/llms.txt`). Это «инструкция для ИИ-поиска» — что компания делает и какие разделы. GEO-фишка, которой у конкурентов обычно нет.
- **`sitemap.xml` + `image-sitemap.xml`** — в WP карту обычно генерит SEO-плагин; эти файлы как образец структуры. После запуска отправить карту в Яндекс.Вебмастер и Google Search Console.
- **`bosfor-2026-7f4a9c2e8d1b6a35.txt`** — ключ **IndexNow**, положить в корень. Включить IndexNow в SEO-плагине (Rank Math / поддержка Яндекса) — страницы будут мгновенно отправляться на переиндексацию.

## 5. Изображения
У всех картинок уже проставлен `alt` (188 шт.) — при загрузке в Медиатеку перенести эти alt. Отдавать в WebP, lazy-load.

## 6. Редиректы со старого сайта (сохранить поисковый вес)
Старый сайт и архивный B2B-домен дают вес. Настроить 301-редиректы со старых URL на новые (плагин Redirection). Карта соответствий — в репозитории проекта (`BOSFOR_REDIRECT_MAP.csv`, `BOSFOR_NEXT_REDIRECTS.json`). B2B-домен после переноса контента закрыть от индексации.

## 7. После запуска (это уже не вёрстка)
- Подключить Яндекс.Вебмастер и Google Search Console, отправить sitemap, проверить индексацию.
- Карточки Яндекс Бизнес и Google Business Profile с тем же NAP (тел. 8-800-101-17-41, Пятигорск, ул. Ермолова, 14).
- Core Web Vitals на мобильном — зелёные (PageSpeed Insights).

## Чек-лист «SEO/GEO не потеряно»
- [ ] title + description из CSV проставлены на всех страницах
- [ ] слаги стоят по ЧПУ-списку (не английские имена файлов)
- [ ] schema на месте, rich-results тест зелёный
- [ ] robots.txt с ИИ-краулерами в корне
- [ ] llms.txt в корне доступен
- [ ] sitemap отправлен в Вебмастер и GSC
- [ ] IndexNow подключён, ключевой файл в корне
- [ ] alt у изображений перенесены
- [ ] 301-редиректы со старого сайта настроены
