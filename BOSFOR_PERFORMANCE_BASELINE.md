# БОСФОР — лабораторный baseline производительности

Дата: 19 июня 2026 года

Профиль: мобильный экран 390×844, задержка 150 мс, загрузка 1,6 Мбит/с,
четырёхкратное замедление CPU.

| Страница | LCP | CLS | Передано |
|---|---:|---:|---:|
| Главная | 1,93 с | 0 | 462 КБ |
| Частным клиентам | 0,34 с | 0 | 163 КБ |
| Застройщикам | 0,26 с | 0 | 304 КБ |

Главный LCP-элемент — изображение первого экрана. Для него подключён AVIF с WebP
fallback, responsive-наборами и preload.

Критические начертания переведены из OTF в WOFF2. Их суммарный вес снижен с
180,3 до 70,2 КБ; OTF оставлены только как fallback.

## Лабораторная задержка взаимодействий

Профиль: мобильный экран 390×844, четырёхкратное замедление CPU.

| Сценарий | Худшее взаимодействие |
|---|---:|
| Открытие мобильного меню | 24 мс |
| Закрытие по Escape | 24 мс |
| Фильтр проектов | 48 мс |
| Фокус поля формы | 176 мс |
| Невалидная отправка | 48 мс |

Худший результат — 176 мс. Это Event Timing baseline, а не полевой INP.

Это лабораторный baseline прототипа, а не итоговая оценка Core Web Vitals.
Приёмка LCP, INP и CLS выполняется на staging и после запуска по данным реальных
пользователей на 75-м перцентиле.

Воспроизводимый замер: `measure_performance.js`.
